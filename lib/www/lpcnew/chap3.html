Kapitel 3 - Variablen und Datentypen

Der Begriff einer Variablen soll hier sehr einfach erläutert werden. Vereinfacht gesagt sind Variablen Speichermöglichkeiten im Programm für verschiedene Datenwerte (zum Beispiel einen Zähler der Stadteinwohner oder die Namen sämtlicher Gildenmitglieder).

Eine Variable bekommt einen Namen und einen Datentyp zugewiesen, bevor sie verwendet werden kann. Dies kann global geschehen, aber auch lokal.

Global bedeutet, die Variable (und vor allem ihr Inhalt) ist im gesamten LPC-Programm jeder Funktion bekannt und somit kann auch der Inhalt der Variablen verändert werden. Globale Variablen werden außerhalb des Funktionsblocks angemeldet.

Lokale Variablen werden innerhalb einer Funktion deklariert und sind auch nur dieser Funktion bekannt. Wird eine lokale Variable mit dem gleichen Namen einer globalen versehen, so überdeckt sie innerhalb der Funktion die globale, so das Zugriffe auf den Inhalt immer den Wert der lokalen Variable erhalten. Nach Beenden der Funktion ist der Inhalt der Variablen und ihr Name vergessen.

Wann global, wozu lokal?
Man mag sich fragen warum man überhaupt lokale Variablen einsetzen sollte, da die globalen einfacher zu handhaben sind (da sie im gesamten Programm verfügbar und bekannt sind).

Zum einen erhöht es die Übersicht wenn Variablen nur dort angemeldet werden wo man sie auch benutzt, zum anderen sind globale Variablen nur erforderlich wenn bestimmte Daten für längere Zeit gespeichert werden sollen.

Nach der Abarbeitung einer Funktion werden die dort angemeldeten lokalen Variablen 'vergessen', womit auch wieder der Vorteil deutlich wird: globale Daten belegen ständig Speicher, lokale nur während ihrer Nutzung.

Beispiele zur Deklaration.

Wie schon erwähnt muß eine Variable vor ihrer Benutzung angemeldet werden damit sie dem Programm bekannt ist.

Bei der Anmeldung (oder auch Deklaration) bekommt die Variable ihren Namen und Datentyp. Es existieren die folgenden Datentypen in LPC:

  • int (ganzzahlige numerische Werte)
  • float (Fließkommazahlen, nicht bei jedem Interpreter)
  • string (Zeichenketten)
  • object (Spezieller Zeiger auf ein LPC Programm)
  • mapping (Eine Daten-Tabelle mit verschiedenen Zugriffsindizes)
  • mixed (Kann jeden der obigen Datentypen enthalten)
  • void (Leerer Datentyp, ist nur für Funktionen verwendbar nicht aber für Variablen)
